# PRP: RPC Operator v0.1 PoC — CRD + Minimaler Operator + Hello-World Pipeline

**Scope:** First milestone from `ARCHITECTURE.md` §9. End state: a `kubectl apply -f pipeline.yaml` produces a running Redpanda Connect Pod that emits visible output to `kubectl logs`.

**Out of scope:** Web UI, Backend API, multi-replica pipelines, metrics/log streaming, RBAC beyond defaults, HA operator. Those are v0.2+.

---

## Goal

A Go-based Kubernetes operator (built with kubebuilder v4) that:

1. Defines a `Pipeline` CRD at `rpc.operator.io/v1alpha1`.
2. Reconciles each `Pipeline` CR by:
   - Rendering a Redpanda Connect `pipeline.yaml` from `spec.input`/`spec.processors`/`spec.output`.
   - Applying a `ConfigMap` (`<name>-config`, key `pipeline.yaml`).
   - Applying a `Pod` (`<name>`) running `docker.redpanda.com/redpandadata/connect:4`, mounting the ConfigMap.
   - Setting OwnerReferences so children GC on CR delete.
   - Managing a `rpc.operator.io/finalizer` for graceful cleanup.
   - Updating `status.phase` (`Pending`/`Running`/`Failed`) and `status.podName`.
3. Ships with one envtest spec verifying the create→children→delete cycle.
4. Ships with a sample CR that, when applied, produces 5 lines of `HELLO WORLD` in `kubectl logs`.

---

## Why

- **Validates the operator shape** before investing in UI/API. If the controller can't reliably manage one Pod per CR, nothing else matters.
- **Establishes the CRD vocabulary** that all later layers (UI, API, multi-replica) will speak.
- **Demonstrates the deployment story** end-to-end: `kubectl apply` → reconcile → running pipeline → visible output.

### Known Gotchas

- **CRITICAL — `stdin` input EOFs in K8s:** A Pod has no interactive FD 0. Connect's `stdin` input reads, hits EOF, treats input as exhausted, exits 0 → CrashLoopBackOff under `restartPolicy: Always`. The sample CR and any test fixtures MUST use `generate` input (`mapping: 'root = "hello world"'`, `interval: 1s`, optionally `count: 5`). The brief's stdin example is conceptual only.
- **CRITICAL — Pod spec is largely immutable:** `controllerutil.CreateOrUpdate` will fail on most Pod-spec mutations after creation. Guard the mutate-fn with `if pod.CreationTimestamp.IsZero()` and only set `pod.Spec` on first creation. For v0.1, do not implement spec-change detection; if the user edits the CR, the existing Pod stays as-is. (Roadmap: v0.2 adds delete-and-recreate-on-spec-change.)
- **CRITICAL — Server-Side Apply not used:** `controllerutil.CreateOrUpdate` is fine for v0.1. SSA is out of scope.
- **CRITICAL — RBAC for `pipelines/finalizers` and `pipelines/status`:** the kubebuilder marker for the controller MUST include both, or finalizer writes / status updates fail with 403 in-cluster. Easy to miss because `make run` (out-of-cluster) uses the kubeconfig user and bypasses controller RBAC.
- **CRITICAL — Default config path is `/connect.yaml`:** the image's default `CMD` is `["run", "/connect.yaml"]`. Either mount the ConfigMap to `/connect.yaml` (and omit `args`) OR mount to `/etc/rpc/pipeline.yaml` and override `args: ["run", "/etc/rpc/pipeline.yaml"]`. The PRP picks the second form for clarity.
- **Image runs as UID 10001 non-root** by default. `readOnlyRootFilesystem: true` works. Do **not** set `runAsUser: 0`.
- **Healthcheck endpoints exist on `:4195`** — `/ping` (liveness, always 200), `/ready` (readiness, 200 only when all I/O connected). Wire both probes.
- **Domain naming:** kubebuilder `--domain operator.io --group rpc` produces API group `rpc.operator.io`. Do **not** invert (`--domain rpc.operator.io --group ...`) — that produces a different group name.

package v1alpha1

import (
    metav1 "k8s.io/apimachinery/pkg/apis/meta/v1"
    "k8s.io/apimachinery/pkg/runtime"
)

// ComponentSpec describes one Redpanda Connect component (input, processor, or output).
// Free-form config matches the heterogeneous schemas of RPC components.
type ComponentSpec struct {
    // Type is the RPC component name, e.g. "generate", "kafka", "mapping", "stdout".
    // +kubebuilder:validation:Required
    Type string `json:"type"`

    // Config is passed verbatim as the body of the component in the rendered YAML.
    // +kubebuilder:pruning:PreserveUnknownFields
    // +kubebuilder:validation:Schemaless
    Config runtime.RawExtension `json:"config,omitempty"`
}

type PipelineSpec struct {
    // +kubebuilder:validation:Required
    Input ComponentSpec `json:"input"`

    Processors []ComponentSpec `json:"processors,omitempty"`

    // +kubebuilder:validation:Required
    Output ComponentSpec `json:"output"`

    // +kubebuilder:default=1
    // +kubebuilder:validation:Minimum=1
    // +kubebuilder:validation:Maximum=1
    // v0.1: only single-replica pipelines. Multi-replica is v0.4+.
    Replicas int32 `json:"replicas,omitempty"`

    // +kubebuilder:default="docker.redpanda.com/redpandadata/connect:4"
    Image string `json:"image,omitempty"`
}

// +kubebuilder:validation:Enum=Pending;Running;Failed;Stopped
type PipelinePhase string

const (
    PhasePending PipelinePhase = "Pending"
    PhaseRunning PipelinePhase = "Running"
    PhaseFailed  PipelinePhase = "Failed"
    PhaseStopped PipelinePhase = "Stopped"
)

type PipelineStatus struct {
    Phase              PipelinePhase      `json:"phase,omitempty"`
    PodName            string             `json:"podName,omitempty"`
    ObservedGeneration int64              `json:"observedGeneration,omitempty"`
    Conditions         []metav1.Condition `json:"conditions,omitempty"`
}

// +kubebuilder:object:root=true
// +kubebuilder:subresource:status
// +kubebuilder:printcolumn:name="Phase",type=string,JSONPath=`.status.phase`
// +kubebuilder:printcolumn:name="Pod",type=string,JSONPath=`.status.podName`
// +kubebuilder:printcolumn:name="Age",type=date,JSONPath=`.metadata.creationTimestamp`
type Pipeline struct {
    metav1.TypeMeta   `json:",inline"`
    metav1.ObjectMeta `json:"metadata,omitempty"`

    Spec   PipelineSpec   `json:"spec,omitempty"`
    Status PipelineStatus `json:"status,omitempty"`
}

// +kubebuilder:object:root=true
type PipelineList struct {
    metav1.TypeMeta `json:",inline"`
    metav1.ListMeta `json:"metadata,omitempty"`
    Items           []Pipeline `json:"items"`
}

### Task 3: Implement the YAML renderer

CREATE `internal/controller/pipeline_render.go`:

```go
package controller

import (
    "encoding/json"
    "fmt"

    "sigs.k8s.io/yaml"

    rpcv1alpha1 "github.com/insidegreen/rpc-operator-claude/api/v1alpha1"
)

// renderPipelineYAML produces a Redpanda Connect config from a PipelineSpec.
// Output shape:
//   input: { <type>: { ...config... } }
//   pipeline: { processors: [ { <type>: { ...config... } }, ... ] }
//   output: { <type>: { ...config... } }
//   http: { enabled: true, address: 0.0.0.0:4195 }
func renderPipelineYAML(spec *rpcv1alpha1.PipelineSpec) (string, error) {
    inputBlock, err := componentBlock(&spec.Input)
    if err != nil {
        return "", fmt.Errorf("input: %w", err)
    }
    outputBlock, err := componentBlock(&spec.Output)
    if err != nil {
        return "", fmt.Errorf("output: %w", err)
    }
    procBlocks := make([]map[string]any, 0, len(spec.Processors))
    for i, p := range spec.Processors {
        b, err := componentBlock(&p)
        if err != nil {
            return "", fmt.Errorf("processors[%d]: %w", i, err)
        }
        procBlocks = append(procBlocks, b)
    }

    doc := map[string]any{
        "input":    inputBlock,
        "pipeline": map[string]any{"processors": procBlocks},
        "output":   outputBlock,
        "http":     map[string]any{"enabled": true, "address": "0.0.0.0:4195"},
    }

    out, err := yaml.Marshal(doc)
    if err != nil {
        return "", err
    }
    return string(out), nil
}

func componentBlock(c *rpcv1alpha1.ComponentSpec) (map[string]any, error) {
    var cfg any = map[string]any{}
    if len(c.Config.Raw) > 0 {
        if err := json.Unmarshal(c.Config.Raw, &cfg); err != nil {
            return nil, fmt.Errorf("config not valid JSON: %w", err)
        }
    }
    return map[string]any{c.Type: cfg}, nil
}
```

Notes:
- Uses `sigs.k8s.io/yaml` (already a transitive dep via controller-runtime) — produces clean YAML from JSON-tagged structs.
- `runtime.RawExtension.Raw` is JSON; `sigs.k8s.io/yaml` round-trips JSON→YAML correctly.
- The `http` block is required for liveness/readiness probes to work. Don't skip it.

package controller

import (
    corev1 "k8s.io/api/core/v1"
    "k8s.io/apimachinery/pkg/api/resource"
    "k8s.io/utils/ptr"
)

const (
    configMountPath = "/etc/rpc"
    configFileName  = "pipeline.yaml"
    httpPort        = 4195
    rpcUID          = int64(10001)
)

func buildPodSpec(cmName, image string) corev1.PodSpec {
    if image == "" {
        image = "docker.redpanda.com/redpandadata/connect:4"
    }
    return corev1.PodSpec{
        RestartPolicy:                 corev1.RestartPolicyAlways,
        TerminationGracePeriodSeconds: ptr.To[int64](30),
        SecurityContext: &corev1.PodSecurityContext{
            RunAsNonRoot: ptr.To(true),
            RunAsUser:    ptr.To(rpcUID),
            FSGroup:      ptr.To(rpcUID),
            SeccompProfile: &corev1.SeccompProfile{
                Type: corev1.SeccompProfileTypeRuntimeDefault,
            },
        },
        Containers: []corev1.Container{{
            Name:  "connect",
            Image: image,
            Args:  []string{"run", configMountPath + "/" + configFileName},
            Ports: []corev1.ContainerPort{
                {Name: "http", ContainerPort: httpPort, Protocol: corev1.ProtocolTCP},
            },
            LivenessProbe: &corev1.Probe{
                ProbeHandler: corev1.ProbeHandler{HTTPGet: &corev1.HTTPGetAction{
                    Path: "/ping", Port: intStrFromString("http"),
                }},
                InitialDelaySeconds: 5, PeriodSeconds: 10,
            },
            ReadinessProbe: &corev1.Probe{
                ProbeHandler: corev1.ProbeHandler{HTTPGet: &corev1.HTTPGetAction{
                    Path: "/ready", Port: intStrFromString("http"),
                }},
                InitialDelaySeconds: 2, PeriodSeconds: 5,
            },
            SecurityContext: &corev1.SecurityContext{
                AllowPrivilegeEscalation: ptr.To(false),
                ReadOnlyRootFilesystem:   ptr.To(true),
                Capabilities:             &corev1.Capabilities{Drop: []corev1.Capability{"ALL"}},
            },
            Resources: corev1.ResourceRequirements{
                Requests: corev1.ResourceList{
                    corev1.ResourceCPU:    resource.MustParse("50m"),
                    corev1.ResourceMemory: resource.MustParse("64Mi"),
                },
                Limits: corev1.ResourceList{
                    corev1.ResourceCPU:    resource.MustParse("500m"),
                    corev1.ResourceMemory: resource.MustParse("256Mi"),
                },
            },
            VolumeMounts: []corev1.VolumeMount{
                {Name: "cfg", MountPath: configMountPath, ReadOnly: true},
            },
        }},
        Volumes: []corev1.Volume{{
            Name: "cfg",
            VolumeSource: corev1.VolumeSource{
                ConfigMap: &corev1.ConfigMapVolumeSource{
                    LocalObjectReference: corev1.LocalObjectReference{Name: cmName},
                    Items: []corev1.KeyToPath{{Key: configFileName, Path: configFileName}},
                },
            },
        }},
    }
}

### Task 5: Implement the reconciler

MODIFY `internal/controller/pipeline_controller.go`. Replace the scaffolded `Reconcile` with this body:

```go
package controller

import (
    "context"
    "fmt"

    corev1 "k8s.io/api/core/v1"
    apierrors "k8s.io/apimachinery/pkg/api/errors"
    metav1 "k8s.io/apimachinery/pkg/apis/meta/v1"
    "k8s.io/apimachinery/pkg/runtime"
    ctrl "sigs.k8s.io/controller-runtime"
    "sigs.k8s.io/controller-runtime/pkg/client"
    "sigs.k8s.io/controller-runtime/pkg/controller/controllerutil"
    logf "sigs.k8s.io/controller-runtime/pkg/log"

    rpcv1alpha1 "github.com/insidegreen/rpc-operator-claude/api/v1alpha1"
)

const finalizerName = "rpc.operator.io/finalizer"

type PipelineReconciler struct {
    client.Client
    Scheme *runtime.Scheme
}

// +kubebuilder:rbac:groups=rpc.operator.io,resources=pipelines,verbs=get;list;watch;create;update;patch;delete
// +kubebuilder:rbac:groups=rpc.operator.io,resources=pipelines/status,verbs=get;update;patch
// +kubebuilder:rbac:groups=rpc.operator.io,resources=pipelines/finalizers,verbs=update
// +kubebuilder:rbac:groups=core,resources=pods,verbs=get;list;watch;create;update;patch;delete
// +kubebuilder:rbac:groups=core,resources=configmaps,verbs=get;list;watch;create;update;patch;delete
// +kubebuilder:rbac:groups=core,resources=events,verbs=create;patch
func (r *PipelineReconciler) Reconcile(ctx context.Context, req ctrl.Request) (ctrl.Result, error) {
    log := logf.FromContext(ctx)

    var pipe rpcv1alpha1.Pipeline
    if err := r.Get(ctx, req.NamespacedName, &pipe); err != nil {
        return ctrl.Result{}, client.IgnoreNotFound(err)
    }

    // Deletion path: finalizer cleanup, then exit.
    if !pipe.DeletionTimestamp.IsZero() {
        if controllerutil.ContainsFinalizer(&pipe, finalizerName) {
            // OwnerReferences GC the children; nothing external to clean up in v0.1.
            controllerutil.RemoveFinalizer(&pipe, finalizerName)
            if err := r.Update(ctx, &pipe); err != nil {
                return ctrl.Result{}, err
            }
        }
        return ctrl.Result{}, nil
    }

    // Add finalizer on first sight, then requeue for a fresh fetch.
    if controllerutil.AddFinalizer(&pipe, finalizerName) {
        if err := r.Update(ctx, &pipe); err != nil {
            return ctrl.Result{}, err
        }
        return ctrl.Result{Requeue: true}, nil
    }

    // Render config.
    yamlStr, err := renderPipelineYAML(&pipe.Spec)
    if err != nil {
        log.Error(err, "render failed")
        return r.markFailed(ctx, &pipe, "RenderError", err.Error())
    }

    // Apply ConfigMap.
    cm := &corev1.ConfigMap{ObjectMeta: metav1.ObjectMeta{
        Name: pipe.Name + "-config", Namespace: pipe.Namespace,
    }}
    if _, err := controllerutil.CreateOrUpdate(ctx, r.Client, cm, func() error {
        cm.Data = map[string]string{"pipeline.yaml": yamlStr}
        return controllerutil.SetControllerReference(&pipe, cm, r.Scheme)
    }); err != nil {
        return ctrl.Result{}, fmt.Errorf("apply configmap: %w", err)
    }

    // Apply Pod (only set spec on creation; pod spec is mostly immutable).
    pod := &corev1.Pod{ObjectMeta: metav1.ObjectMeta{
        Name: pipe.Name, Namespace: pipe.Namespace,
    }}
    if _, err := controllerutil.CreateOrUpdate(ctx, r.Client, pod, func() error {
        if pod.CreationTimestamp.IsZero() {
            pod.Spec = buildPodSpec(cm.Name, pipe.Spec.Image)
            pod.Labels = map[string]string{
                "rpc.operator.io/pipeline": pipe.Name,
            }
        }
        return controllerutil.SetControllerReference(&pipe, pod, r.Scheme)
    }); err != nil {
        return ctrl.Result{}, fmt.Errorf("apply pod: %w", err)
    }

    // Status update.
    desired := derivePhase(pod)
    if pipe.Status.Phase != desired ||
        pipe.Status.PodName != pod.Name ||
        pipe.Status.ObservedGeneration != pipe.Generation {
        pipe.Status.Phase = desired
        pipe.Status.PodName = pod.Name
        pipe.Status.ObservedGeneration = pipe.Generation
        if err := r.Status().Update(ctx, &pipe); err != nil {
            if apierrors.IsConflict(err) {
                return ctrl.Result{Requeue: true}, nil
            }
            return ctrl.Result{}, err
        }
    }

    return ctrl.Result{}, nil
}

func derivePhase(pod *corev1.Pod) rpcv1alpha1.PipelinePhase {
    switch pod.Status.Phase {
    case corev1.PodRunning:
        return rpcv1alpha1.PhaseRunning
    case corev1.PodFailed:
        return rpcv1alpha1.PhaseFailed
    case corev1.PodSucceeded:
        return rpcv1alpha1.PhaseStopped
    default:
        return rpcv1alpha1.PhasePending
    }
}

func (r *PipelineReconciler) markFailed(
    ctx context.Context, pipe *rpcv1alpha1.Pipeline, reason, msg string,
) (ctrl.Result, error) {
    pipe.Status.Phase = rpcv1alpha1.PhaseFailed
    pipe.Status.Conditions = []metav1.Condition{{
        Type:    "Ready",
        Status:  metav1.ConditionFalse,
        Reason:  reason,
        Message: msg,
        LastTransitionTime: metav1.Now(),
    }}
    if err := r.Status().Update(ctx, pipe); err != nil {
        return ctrl.Result{}, err
    }
    return ctrl.Result{}, nil
}

func (r *PipelineReconciler) SetupWithManager(mgr ctrl.Manager) error {
    return ctrl.NewControllerManagedBy(mgr).
        For(&rpcv1alpha1.Pipeline{}).
        Owns(&corev1.ConfigMap{}).
        Owns(&corev1.Pod{}).
        Complete(r)
}
```

## Risks & Edge Cases

- **Status-update conflict storms.** Each Pod-status change triggers a reconcile, which writes Pipeline status, which can collide with the K8s API's resourceVersion. Mitigation: the code returns `Requeue: true` on `IsConflict` rather than erroring.
- **Spec drift after creation.** If a user `kubectl edit pipeline/foo` to change the input type, the existing Pod ignores the change (CreationTimestamp guard). The ConfigMap *will* update, but Connect doesn't hot-reload by default. Document this as a v0.1 limitation in the sample CR comment. v0.2 adds detect-and-recreate.
- **Pod stuck Pending due to image-pull.** First reconcile sees `PodPending`; status reads `Pending` indefinitely. Acceptable for v0.1; v0.5 will surface pull errors as a Condition.
- **`generate` with `count: N` exits.** Pod transitions to `Succeeded` → status `Stopped`. With `restartPolicy: Always` the kubelet will restart the container, so the user sees repeating output. For a one-shot demo this is fine; for the smoke test, accept either `Running` or `Stopped`.
- **`runtime.RawExtension` round-trip.** When `Config` is empty, `Raw` may be `nil` or `[]byte("null")`. The renderer treats both as empty — covered by the `TestRenderPipelineYAML_EmptyConfig` case.
- **CRD schema validation surprises.** `+kubebuilder:pruning:PreserveUnknownFields` + `+kubebuilder:validation:Schemaless` on `ComponentSpec.Config` is required; without them, kube-apiserver strips unknown keys before they reach the operator. Test by applying a CR with a deeply nested `config` and reading it back — fields must round-trip.
- **Finalizer wedging on uninstall.** If you `kubectl delete crd pipelines.rpc.operator.io` while a CR exists, the finalizer prevents deletion and the CRD removal hangs. Document `kubectl patch pipeline/<n> -p '{"metadata":{"finalizers":[]}}' --type=merge` as the escape hatch in the README (out of scope for code, in scope for the runbook).
- **envtest does not run the GC controller.** Assertions about ConfigMap/Pod *deletion* via OwnerRefs will fail in envtest. Assert ownerRef *presence* and the Pipeline finalizer cycle instead. Real GC verification belongs in the kind smoke test.

## Anti-Patterns to Avoid

- **Don't use Server-Side Apply** for v0.1. `controllerutil.CreateOrUpdate` is the kubebuilder norm at v0.23 and adequate.
- **Don't mutate Pod spec on update.** The CreationTimestamp guard exists for a reason; `kube-apiserver` will reject most field changes.
- **Don't add multi-replica logic.** `replicas` is capped at 1 in the schema; v0.4 will lift this with a Deployment per pipeline + per-input scaling rules.
- **Don't introduce a `client-go/fake` controller test** instead of envtest. The fake client doesn't enforce OwnerReferences, doesn't simulate conflicts, and silently accepts illegal patches.
- **Don't skip `make manifests` before commit.** The CRD/RBAC YAML must match the Go markers; PR review will reject drift.
- **Don't add log/metric streaming, UI, or API endpoints.** Those are v0.2 / v0.3 / v0.4 milestones.
- **Don't write your own deepcopy.** `make generate` produces correct deepcopies for `runtime.RawExtension` fields; hand-rolled versions break.
- **Don't put an `omitempty` on `ComponentSpec.Type`.** It's required; the schema enforces it.
- **Don't hardcode the image with `:latest`.** Pin to `:4` (major) at minimum so the PoC isn't broken by an unrelated upstream release.
